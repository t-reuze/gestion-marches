Travailler sur le projet — Git pour débutants
💡
Git, c'est quoi ? Un système de sauvegarde intelligent. Chaque personne travaille dans son coin (sa branche), puis on regroupe le travail quand c'est prêt — sans écraser celui des autres.

Étape 0 — Installation (une seule fois)

A

Vérifier que Git est installé

Ouvre VS Code → terminal (Ctrl+J) → tape :

git --version

Tu dois voir git version 2.x.x. Sinon, télécharge Git sur git-scm.com.

B

Configurer ton identité

git config --global user.name "Prénom Nom"
git config --global user.email "ton@email.com"
C

Télécharger le projet

git clone https://github.com/t-reuze/gestion-marches.git

Puis ouvre le dossier gestion-marches dans VS Code.

Le quotidien — 5 commandes

1

Se mettre à jour (chaque matin)

git checkout feature/notation   # va sur ta branche
git pull                          # récupère les dernières modifs
2

Travailler normalement

Modifie tes fichiers dans VS Code comme d'habitude. Rien ne change.

3

Sauvegarder son travail

git add .
git commit -m "feat: description de ce que j'ai fait"

Le message doit décrire ce que tu as fait, pas comment. Exemples :
feat: ajout filtre fournisseur  ·  fix: correction calcul lots

4

Envoyer sur GitHub

git push
5

Demander une validation (Pull Request)

Quand ta tâche est terminée :

  • Va sur github.com/t-reuze/gestion-marches
  • Clique sur la bannière jaune "Compare & pull request"
  • Remplis le titre, assigne t-reuze en reviewer
  • Clique "Create pull request"

Aide-mémoire — les commandes essentielles

git pullRécupérer les dernières modifications
git statusVoir ce que j'ai modifié
git add .Sélectionner tous mes fichiers modifiés
git commit -m "message"Créer un point de sauvegarde
git pushEnvoyer sur GitHub
git branchVoir sur quelle branche je suis

Problèmes fréquents

ProblèmeSolution
Je ne sais plus sur quelle branche je suis Regarde en bas à gauche de VS Code, ou tape git branch
J'ai modifié un fichier par erreur git restore nom-du-fichier.jsx ⚠️ efface définitivement
J'ai un conflit Ne touche à rien — appelle t-reuze. Les conflits se règlent à deux.
Git demande un mot de passe C'est ton mot de passe GitHub. Si ça bloque, contacter t-reuze.
🚫
Règle absolue : On ne pousse jamais directement sur main. Toujours passer par une Pull Request. Si GitHub te bloque, c'est normal — c'est fait exprès.

Travailler avec Claude Code

Claude Code — l'assistant IA intégré à VS Code

Claude Code est un assistant développeur qui comprend l'ensemble du projet. Il peut lire, modifier et créer des fichiers, exécuter des commandes, et expliquer le code existant.

Lecture de code
Il lit les fichiers du projet et comprend le contexte avant de répondre
Modification de fichiers
Il peut éditer directement des composants React, corriger des bugs
Commandes terminal
Il peut lancer git, npm et d'autres commandes
Explications
Il explique n'importe quelle partie du code en langage simple

Comment accéder à Claude Code

1

Depuis VS Code

Claude Code est disponible via l'extension VS Code. Clique sur l'icône Claude dans la barre latérale gauche, ou utilise le raccourci Ctrl+Shift+C.

2

Depuis le terminal

Ouvre un terminal dans le dossier du projet et tape :

claude
3

Depuis claude.ai/code

Accessible aussi via navigateur sur claude.ai/code avec ton compte Anthropic.

Comment bien formuler ses demandes

✅
Donne du contexte. Plus ta demande est précise, meilleure est la réponse.
Claude lit les fichiers du projet automatiquement — pas besoin de tout recopier.
Demande vagueDemande efficace
« Ça marche pas » « La page Notation affiche une erreur quand il n'y a pas de données — comment corriger ? »
« Fais quelque chose de joli » « Dans KpiCard.jsx, ajoute une icône à gauche du titre. Je veux que ce soit discret. »
« Ajoute une fonctionnalité » « Dans la page Reporting, ajoute un filtre par statut de marché au-dessus du tableau. »

Exemples de demandes utiles

→

Comprendre du code

« Explique-moi comment fonctionne le fichier NotationContext.jsx »
« Qu'est-ce que useParams() fait dans cette page ? »

→

Corriger un bug

« La grille de notation s'affiche vide quand je sélectionne ce marché — regarde pourquoi »
« J'ai cette erreur dans la console : [coller l'erreur] — comment la corriger ? »

→

Ajouter une fonctionnalité

« Dans Notation.jsx, ajoute un bouton "Exporter en PDF" qui télécharge la grille »
« Ajoute une colonne "Commentaire" dans le tableau des fournisseurs »

→

Aide Git

« J'ai un conflit sur ce fichier, aide-moi à le résoudre »
« Je veux créer une nouvelle branche pour ajouter l'export PDF — comment faire ? »

⚠️
Toujours relire avant d'accepter. Claude peut se tromper sur la logique métier. Il ne connaît pas les règles Unicancer — toi oui. Lis ce qu'il propose avant de valider les modifications.

Bonnes pratiques avec Claude Code

  • Une tâche à la fois. « Corrige ce bug » plutôt que « Corrige ce bug et refais tout le design ».
  • Travaille sur ta branche. Avant de demander à Claude de modifier des fichiers, vérifie que tu es bien sur ta branche (git branch).
  • Commite après chaque modification. Si Claude fait une modif et que ça casse quelque chose, git restore . annule tout.
  • Tu peux lui demander d'expliquer. Claude peut justifier chaque ligne qu'il écrit — n'hésite pas à demander « pourquoi tu fais ça comme ça ? »
